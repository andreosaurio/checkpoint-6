¿Para qué usamos clases en Python?

Las clases en Python son una herramienta para crear objetos que pueden contener atributos y métodos, es decir, unas características y unos comportamientos específicos. Las clases nos permiten agrupar datos y métodos relacionados, facilitando la organización y la reutilización del código. 

Para crear una clase usamos la palabra reservada class. La convenciones de nomenclatura de Python conocidas como PEP 8 recomiendan el uso de CamelCase para nombrar clases. Además, es conveniente elegir nombres que sean descriptivos tanto para las clases como para los atributos.

Ejemplo:

class Invoice:
    def greeting(self):
        return 'Hi there'
inv_one = Invoice()
print(inv_one.greeting())

En el anterior ejemplo la clase Invoice tiene un metodo llamado greeting que devuelve el string Hi there. El parámetro self se usa para referirse a la instancia de la clase. Es necesario incluir 'self' como primer parámetro en cada método de clase.
Despúes de definir la clase, se crea una instancia de ésta llamada inv_one. 
Por último, se llama al método greeting() de la instancia inv_one y se imprime el resultado.

Sin embargo, el ejemplo anterior es la forma más simple y no sería verdaderamente útil en cuanto a utilización en una aplicación real. Para comprender las clases es necesario entender a su vez la función especial __init__(), también conocida como dunder init. Esta función reservada es el constructor y se ejecuta automáticamente cuando la clase se instancia. Lo veremos más adelante.

Otros subtemas de interés relacionados con las clases son otros métodos dunder, el encapsulamiento, la herencia o el polimorfismo.

¿Qué método se ejecuta automáticamente cuando se crea una instancia de una clase?

El método que se ejecuta automáticamente cuando se crea una instancia de una clase se llama __init__. Este método se encarga de inicializar los atributos de la instancia. 

Ejemplo:

class Persona:
    def __init__(self, nombre, apellido):
        self.nombre = nombre
        self.apellido = apellido

persona1 = Persona('Andrea', 'Fernández')

print(persona1.nombre) # Salida : Andrea
print(persona1.apellido) # Salida: Fernández

En la primera línea se define la clase Persona. 

Después se define el constructor a través del método __init__. Este método se va a ejecutar automáticamente cada vez que se cree una nueva instancia de la clase Persona. Toma tres parámetros:

  • self: es necesario en todos los métodos de clase Python y hace referencia a la instancia de la clase.
  • nombre y apellido: son los atributos que se inicializarán para cada instancia de la clase.

En la siguiente línea se crea una nueva instancia de la clase Persona llamada persona1. Así es como se llama automáticamente al método __init__ para inicializar los atributos nombre y apellido con los valores proporcionados Andrea y Fernández. 
Después se imprimen los valores de los atributos.

¿Cuáles son los tres verbos de API?

Los verbos de API son instrucciones estandarizadas que indican la acción que una aplicación quiere realizar sobre un recurso en otra aplicación. Estos verbos se utilizan en APIs Rest, que son el tipo de API más común en la actualidad.

Los tres verbos principales de API son:

  • GET: Obtener datos de un servidor.
  • POST: Enviar datos nuevos al servidor.
  • PUT / PATCH: Para actualizar los datos existentes en el servidor. 

Además de estos verbos básicos, también se pueden usar otros verbos adicionales para acciones específicas, como:

  • DELETE: Se utiliza para eliminar datos.
  • HEAD: Para obtener solamente los encabezados en un servidor, sin el cuerpo completo. 

Hay que ser especialmente precavido al utilizar verbos como PUT o DELETE, ya que pueden modificar o eliminar datos de forma permanente.

¿Es MongoDB una base de datos SQL o NoSQL?

MongoDB es una base de datos NoSQL clasificada de tipo Documental, ya que sus datos se almacenan en documentos JSON.

Las características principales de las bases de datos NoSQL son:

  • No relacionales: No se basan en tablas y relaciones.
  • No estructuradas: Los datos pueden tener diferentes formatos.
  • Consultas específicas: Cada tipo de base de datos NoSQL tiene su propio lenguaje de consulta.

Las ventajas de utilizar MongoDB son las siguientes:

  • Flexibilidad: Almacena datos en documentos JSON, que son flexibles y fáciles de modificar.
  • Escalabilidad: Puede escalar horizontalmente para manejar grandes cantidades de datos.
  • Rendimiento: Es una base de datos rápida y eficiente.

¿Qué es una API?

API significa Interfaz de Programación de APlicaciones (en inglés, Application Programming Interface). Es un conjunto de reglas y definiciones que permiten que dos aplicaciones se comuniquen entre sí.

Las APIs se utilizan para:

  • Obtener datos: Por ejemplo, una aplicación de noticias podría usar una API para obtener datos sobre los últimos artículos.
  • Enviar datos: Una app como Instagram utilizaría una API para enviar las publicaciones de los usuarios.
  • Controlar dispositivos: Una aplicación de domótica podría usar una API para subir o bajar las persianas.

Tipos de APIs:

  • API REST: Es el tipo de API más común. Se basa en el uso de verbos HTTP (GET, POST, PUT, DELETE) para realizar las acciones sobre los recursos.
  • API SOAP: Es la más antigua, utilizando XML para definir los mensajes que se intercambian las aplicaciones.
  • API GraphQL: Es la más reciente, permite que los clientes soliciten datos de forma específica y flexible.

Las APIs se utilizan en una gran variedad de aplicaciones, como aplicaciones web, aplicaciones móviles, juegos y software empresarial. 

Algunos ejemplos de APIs son:

  • Google Maps API: permite obtener información sobre mapas, direcciones y lugares.
  • Twitter API: permite obtener información sobre tweets, usuarios y hastags.
  • Facebook API: permite obtener información sobre usuarios, publicaciones y eventos.

Otros temas de interés en relación con las API son la arquitectura de software, la integración de aplicaciones y el diseño de APIs.

¿Qué es Postman?

Postman es una herramienta de colaboración para el desarrollo de APIs. Postman proporciona una interfaz gráfica de usuario que permite lo siguiente:

  • Enviar solicitudes a APIs: se puede probar diferentes verbos HTTP, encabezados y cuerpos de solicitud.
  • Recibir y analizar respuestas de la API: se puede ver el código de estado, los encabezados y el cuerpo de la respuesta.
  • Documentar APIs: se pueden crear y compartir colecciones de solicitudes y respuestas.
  • Colaborar en equipo: puedes trabajar con otros en las mismas colecciones de API.

Postman es ampliamente utilizado en el desarrollo de software para probar y depurar APIs antes de integrarlas en aplicaciones más grandes. Por lo tanto, es una herramienta útil para desarrolladores.

¿Qué es el polimorfismo?

El polimorfismo es un concepto de la programación orientada a objetos que permite que diferentes objetos respondan al mismo mensaje de diferentes maneras. Dicho de otra manera, dos objetos de diferentes clases pueden tener métodos con el mismo nombre, y los dos pueden ser llamados con el mismo código, dando respuestas diferentes.

En Python el polimorfismo se puede lograr de dos formas:

  • Polimorfismo de inclusión: basado en la herencia.
  • Polimorfismo de sobrecarga: basado en la definición de diferentes métodos con el mismo nombre pero con diferentes parámetros.

Los beneficios de utilizar el polimorfismo son:

  • Flexibilidad: permite escribir código más flexible y reutilizable.
  • Extensibilidad: facilita la creación de nuevas clases que heredan de clases existentes.
  • Mantenimiento: hace que el código sea más fácil de mantener y actualizar.

Ejemplo:

class Animal:
    def hablar(self):
        raise NotImplementedError

class Perro(Animal):
    def hablar(self):
         return 'Guau!'

class Gato(Animal):
    def hablar(self):
        return 'Miau!'

perro = Perro()
gato = Gato()

print(perro.hablar()) # Salida: Guau!
print(gato.hablar()) # Salida: Miau!

En este ejemplo el polimorfismo se representa mediante la capacidad de los objetos de las clases Perro y Gato de comportarse de manera similar aunque sean de diferentes tipos, permitiendo escribir código genérico.
La clase Animal define un método hablar() que levanta la excepción NotImplementedError, sirviendo de interfaz genérica que especifica que cualquier subclase debe implementar su propio método hablar(). Así pues, las clases Perro y Gato sobreescriben el método hablar con sus propias implementaciones.
Al crear una instancia de Perro y Gato y se llama al método hablar() en cada una de ellas, el resultado es diferente para cada objeto, pero el código que realizada la llamada es el mismo. Esto demuestra que los objetos de diferentes tipos pueden ser tratados de manera uniforme a través de la herencia y el polimorfismo en Python.

¿Qué es un método dunder?

Un método dunder (double underscore), también conocido como método mágico o especial, son métodos predefinidos que comienzan y terminan con dos guiones bajos (_ _). Estos métodos se utilizan para implementar funcionalidades especiales en las clases.

Algunos ejemplos de métodos dunder:

  • __init__: se llama al crear una nueva instancia de una clase. También conocido como método constructor.
  • __str__: se utiliza para definir cómo se debe representar la instancia de la clase como una cadena de caracteres (string). Sobre todo se utiliza para debugging.
  • __repr__: es miuy similar al anterior, pero se usa para definir una representación sin ambigüedades, siendo útil al inspeccionar objetos para los desarrolladores.
  • __iter__: se llama para obtener un iterador de un objeto. Un iterador es un objeto que permite recorrer una secuencia de valores uno por uno.
  • __next__: se llama en un interador para obtener el siguiente valor de la secuencia. Al llegar al final de la secuencia se lanza la excepción StopIteration.
  • __len__: se llama para obtener la longitud de una secuencia.
  • __add__: se llama para sumar dos objetos.

Ejemplo:

class MiClase:
    def __init__(self, lista):
        self.lista = lista

    def __iter__(self):
        return iter(self.lista)

    def __next__(self):
        if not self.lista:
            raise StopIteration
        return self.lista.pop(0)

obj = MiClase([1, 2, 3])
for i in obj:    
print(i) # Salida: 1, 2, 3

En este código se crea una instancia de MiClase con una lista [1, 2, 3]. Después se itera sobre esta instancia utilizando un bucle for. En cada iteración se llama al método __next__ de MiClase para obtener el siguiente elemento del iterador, que es uno de los elementos de la lista. El resultado es que se imprime cada elemento de la lista en orden y uno por uno.

¿Qué es un decorador de Python?

Un decorador de Python es una función que recibe otra función como parámetro y devuelve otra función. Se utilizan para modificar el comportamiento de una función sin tener que modificarla directamente.

Los decoradores son útiles para reutilizar código, permiten la modularidad, es decir, separar la lógica de la decoración de la lógica de la función y, además, ayudan a que el código sea más legible y fácil de entender.

Ejemplo:

def propiedad(func):
    def wrapper(self, *args):
        return func(self, *args)
    return property(wrapper)

class Persona:
   def __init__(self, nombre):
        self._nombre = nombre  # Atributo privado

    @propiedad
    def nombre(self):
        return self._nombre

    @nombre.setter
    def nombre(self, nuevo_nombre):
        self._nombre = nuevo_nombre

persona = Persona("Andrea")
print(persona.nombre)  # Salida: Andrea

persona.nombre = "Mikel"
print(persona.nombre)  # Salida: Mikel

En este ejemplo el decorador propiedad toma la función func como argumento y devuelve la función interna wrapper, que se utiliza como getter de la propiedad.
Dentro de la clase Persona aplicamos este decorador al método nombre utilizando la sintaxis @propiedad. Esto crea una propiedad llamada nombre que actúa como getter para el atributo privado _nombre.
Luego se utiliza el decorador @nombre.setter para definir un método setter para la propiedad nombre. El método setter nos permite modificar el valor del atributo _nombre.
Por último se crea una instancia de la clase Persona y se accede y modifica el atributo nombre utilizando el getter y setter definidos.
